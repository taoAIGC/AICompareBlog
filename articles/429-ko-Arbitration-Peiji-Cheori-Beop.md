## 왜 많은 크로스보더 쇼핑몰은 도움말 페이지나 규정 페이지보다 중재 페이지, 판정 페이지, 분쟁 처리 페이지가 먼저 노출될까?

이런 일은 생각보다 흔합니다.

사이트가 분실 보상, 파손 보상, 보험 분쟁, 보상 재심, 고객지원 에스컬레이션을 다루기 시작하면 다음 같은 URL이 생기기 쉽습니다.

- `/arbitration`
- `/claim/arbitration`
- `/dispute/arbitration`
- `/compensation/arbitration`
- `/arbitration/result`
- `/after-sales/arbitration?caseId=20260327029`

그리고 신청 페이지, 증빙 업로드 페이지, 진행 상황 페이지, 결과 페이지, 종료 페이지까지 늘어납니다.

많은 팀은 이렇게 생각합니다.

- 실제 페이지니까 색인돼도 괜찮다
- 사용자도 중재 관련 검색을 한다
- 상태, 증빙, 결론이 있어서 내용이 빈약해 보이지 않는다
- 시스템이 자동 생성했으니 나중에 정리하면 된다

하지만 시간이 지나면 이런 페이지는 대개 이렇게 됩니다.

- 특정 주문과 특정 분쟁에 지나치게 묶인다
- 검색 유입 사용자는 로그인 화면이나 처리 중 화면만 본다
- 파라미터 URL이 계속 늘어난다
- 원래 검색 수요를 받아야 할 공개 설명 페이지는 약한 상태로 남는다

그래서 **중재 페이지를 어떻게 처리할지** 고민한다면 핵심은 이겁니다.

**대부분의 중재 페이지는 공개 검색 랜딩페이지가 아니라 처리 페이지, 분쟁 대응 페이지, 판정 페이지입니다. 운영상 중요할 수는 있지만, 그렇다고 SEO 핵심 페이지가 되는 것은 아닙니다. 먼저 이 URL이 개별 사건을 위한 것인지, 공개 검색 의도에 답하는 것인지부터 나눠야 합니다.**

## 중재 페이지는 실제로 무엇을 해결하는가?

단순히 “최종 결론이 나오는 페이지”만은 아닙니다.

### 1. 핵심 역할은 기존 사용자 사건 처리이지, 공개 검색 수요 대응이 아니다

보통 이런 일을 합니다.

- 사건이 중재 단계로 들어갔는지 보여준다
- 추가 증빙이 필요한지, 심사 중인지, 종료됐는지 알려준다
- 다음에 무엇을 제출해야 하는지 안내한다
- 고객센터, 물류사, 플랫폼, 사용자의 판단을 맞춘다

### 2. 주문 정보와 증빙 자료에 강하게 묶여 있다

예를 들면:

- 주문번호
- 운송장 번호
- 클레임 ID
- 분쟁 사유
- 업로드된 증빙
- 운송사 회신
- 예비 결론과 최종 판정

그래서 보통 공개 색인에 적합한 안정적인 페이지가 되지 않습니다.

### 3. 비즈니스 가치와 SEO 가치는 다르다

검색 사용자가 진짜 알고 싶은 건 보통 이런 것입니다.

- 언제 중재로 가야 하는지
- 어떤 자료가 필요한지
- 얼마나 걸리는지
- 플랫폼 중재와 운송사 검토 차이가 뭔지
- 결과가 마음에 안 들면 어떻게 해야 하는지

이런 수요는 보통 다음 페이지가 더 잘 받습니다.

- 중재 안내 페이지
- 분쟁 FAQ
- 규정 설명 페이지
- 증빙 준비 가이드
- 고객지원 도움말 페이지

## 중재 페이지 처리법: 먼저 이 5가지를 나눠서 보자

### 1. 신청 페이지, 진행 페이지, 결과 페이지에 불과하다면 보통 핵심 SEO 페이지로 만들 필요가 없다

해당 사용자에게는 필요하지만 공개 검색 의도를 만족시키는 경우는 드뭅니다.

### 2. 실제 검색 수요가 “거절 후 어떻게 해야 하나”, “중재에 필요한 자료는 무엇인가”라면 비공개 결과 페이지가 그 역할을 대신하게 두지 말자

검색 의도는 설명이지 특정 사건 기록이 아닙니다.

### 3. arbitration, dispute, ruling, evidence, appeal 같은 경로가 대량으로 생성된다면 기능 URL과 색인 대상 URL을 분리하자

그렇지 않으면 유사 URL이 늘고 크롤링이 낭비되며 민감 정보가 노출될 수 있습니다.

### 4. 공개 설명 페이지와 비공개 사건 상세 페이지는 완전히 분리하자

공개 페이지는 일반 질문에 답하고, 비공개 페이지는 실제 사건을 처리해야 합니다.

### 5. 랭킹을 원하지 않는다면 noindex, 로그인 제한, sitemap, canonical, 첨부 규칙, 내부 링크를 맞춰야 한다

많은 SEO 문제는 페이지 자체보다 기술 신호 충돌에서 발생합니다.

## 중재 페이지에서 자주 보이는 4가지 SEO 실수

### 1. “중재”를 검색하니 사건 상세 페이지도 랭킹할 수 있다고 생각한다

### 2. 중재 페이지, 증빙 페이지, 판정 페이지, 결과 페이지를 한꺼번에 노출한다

### 3. 검색 가치가 있는 설명을 로그인 뒤에 숨긴다

### 4. 색인을 원하지 않는다고 하면서 sitemap, 템플릿, 첨부, 파라미터는 제각각으로 둔다

## 지금 중재 페이지를 점검하려면 이 순서로 보면 된다

### 1단계: 중재 관련 URL 유형을 모두 정리한다

### 2단계: 각 페이지가 개별 사건용인지 공개 질문용인지 판단한다

### 3단계: 실제 검색 수요는 별도의 공개 페이지로 분리한다

예를 들면:

- 중재 가이드
- 분쟁 FAQ
- 책임 판정 규정
- 증빙 준비 가이드
- 처리 기간 설명 페이지

### 4단계: 기술 신호를 통일한다

### 5단계: 단순히 색인 여부만 보지 않는다

정말 봐야 할 것은 올바른 공개 페이지가 더 좋은 트래픽을 받는지입니다.

## 중재 페이지는 색인해야 할까?

제 생각은 단순합니다.

- 개별 분쟁 처리 페이지라면 핵심 SEO 페이지로 보지 말 것
- 신청, 진행, 결과, 판정 페이지라면 보통 랭킹이 필요하지 않음
- 진짜 가치가 규정, 자료, 기간, 기준에 있다면 그 내용을 별도 공개 페이지로 만들 것

그래서 아직도 **중재 페이지를 어떻게 처리할지** 고민한다면 결론은 이겁니다.

**먼저 중재 페이지를 처리 페이지로 보세요. 그다음 정말 색인 가치가 있는 부분만 따로 떼어내면 됩니다. 시스템이 자동 생성했다는 이유만으로 비공개 중재 URL이 검색 결과를 채우게 두지 않는 편이 낫습니다.**
