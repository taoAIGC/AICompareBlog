## 왜 많은 사이트는 404를 발견하면 바로 전부 홈페이지로 보내 버릴까?

이건 정말 자주 보는 패턴입니다.

404 페이지가 있다는 걸 알게 되는 순간, 많은 팀이 이렇게 반응합니다.

- 일단 다 리다이렉트하자
- 사용자가 에러 페이지를 보게 하지 말자
- Google이 404를 보지 못하게 하자
- 전부 홈페이지로 보내자

겉보기에는 빠른 대응처럼 보이지만, 실제로는 SEO 신호를 더 꼬이게 만드는 경우가 많습니다.

404 자체가 곧바로 SEO 문제라는 뜻은 아니기 때문입니다.

진짜 문제는 보통 이런 데 있습니다.

- 사라진 페이지 흔적이 사이트 안에 그대로 남아 있음
- 원래는 대체 페이지가 있는데 404로 방치됨
- 내부 링크가 여전히 존재하지 않는 URL을 가리킴
- sitemap 에 삭제된 페이지가 계속 남아 있음
- 내용은 사실상 없는데 200을 반환해서 soft 404가 됨

그래서 404 페이지를 어떻게 처리해야 하는지 고민할 때, 핵심 결론은 아주 단순합니다.

**모든 404에 리다이렉트가 필요한 것도 아니고, 모든 리다이렉트 목적지가 홈페이지여야 하는 것도 아닙니다.**

먼저 그 페이지가 정말 사라진 건지, 위치만 바뀐 건지, 아니면 URL 오타 문제인지부터 구분해야 합니다.

## 404는 검색엔진에 무엇을 말하고 있을까?

많은 사람은 404를 보면 “사이트가 고장 났다”라고 느낍니다.

하지만 기술적으로 의미는 더 단순합니다.

> 요청한 URL에 지금은 제공할 콘텐츠가 없다.

이게 전부입니다.

404가 나오는 건 웹에서는 전혀 이상한 일이 아닙니다.

예를 들면:

- 오래된 글을 삭제했다
- 상품이 완전히 단종되었다
- 이벤트 페이지가 종료되었다
- 사용자가 주소를 잘못 입력했다
- 외부 사이트가 잘못된 링크를 걸었다

문제는 404가 존재한다는 사실 자체가 아닙니다.

**사이트 전체가 404를 얼마나 일관되게 다루고 있느냐**가 중요합니다.

제가 404 페이지 SEO를 볼 때는 먼저 이 4가지를 확인합니다.

### 1. 정말 사라진 페이지는 404 또는 410을 정확히 반환하는가

없는 페이지라면 없다고 솔직하게 말하는 편이 낫습니다.

### 2. 대체 페이지가 있는 경우, 가장 가까운 페이지로 정확히 보내는가

그냥 열리는 아무 페이지로 보내는 건 의미가 없습니다.

### 3. 사이트 내부가 아직도 죽은 URL로 트래픽을 보내고 있지 않은가

메뉴, 본문 링크, 관련 글, sitemap 이 계속 죽은 URL을 가리키면 404는 마지막에 보이는 증상일 뿐입니다.

### 4. 사용자가 다음 행동을 할 수 있는 길이 남아 있는가

좋은 404 페이지는 탐색이 거기서 끝나지 않게 만들어야 합니다.

## 404 페이지 처리 방법: 저는 먼저 이 5가지 경우로 나눠서 봅니다

### 1. 페이지가 영구적으로 삭제된 것이라면 404나 410으로 두는 편이 낫다

이 경우가 가장 많이 “과하게 수정”됩니다.

예를 들면:

- 더 이상 필요 없는 오래된 공지
- 다시 열릴 일 없는 이벤트 페이지
- 삭제된 테스트 페이지
- 후속 상품도 없는 단종 페이지

이런 경우에는 오히려 깔끔하게

**404 또는 더 명확하게 410을 반환하는 것**

이 좋습니다.

이런 페이지를 전부 홈페이지로 보내 버리면 이런 일이 생깁니다.

- 예전 URL의 주제와 홈페이지 내용이 맞지 않음
- 사용자의 기대가 충족되지 않음
- 관련 없는 URL이 한곳으로 몰림
- 사이트 구조가 대충 만든 것처럼 보임

에러를 없앤 것처럼 보여도, 실제로는 문제를 홈페이지로 밀어 넣었을 뿐인 경우가 많습니다.

---

### 2. 페이지가 이동한 것이라면 올바른 301 리다이렉트를 사용해야 한다

이건 반대 사례입니다.

페이지가 사라진 게 아니라 단지 이렇게 바뀐 것일 수 있습니다.

- URL 구조가 바뀌었다
- 오래된 글이 새 글로 통합되었다
- 상품 페이지가 새 버전으로 교체되었다
- 카테고리 구조가 바뀌었다

이럴 때 예전 URL을 404로 두면 원래 이어받을 수 있었던 신호까지 끊어 버리게 됩니다.

이 경우 기본 원칙은:

**기존 URL을 가장 관련성 높은 새 URL로 301 연결하는 것**

입니다.

여기서 자주 보이는 대충 한 방식은 두 가지입니다.

- 옛 URL을 전부 홈페이지로 보내기
- 옛 URL을 전부 같은 카테고리 페이지로 보내기

오늘 작업은 편할지 몰라도, 나중에 보면 대부분 부자연스럽습니다.

---

### 3. 상품 종료나 대량 삭제 때는 검색 수요가 남아 있는지 먼저 봐야 한다

이 경우는 판단이 조금 더 필요합니다.

예를 들면:

- 판매 종료 상품
- 더 이상 유지하지 않는 필터 페이지
- 비공개 처리한 콘텐츠 시리즈
- 기간이 끝난 랜딩 페이지

이걸 전부 같은 방식으로 처리하는 건 추천하지 않습니다.

저는 먼저 두 가지를 봅니다.

### 그 페이지에 안정적인 유입이 있었는가

있었다면 대충 지워도 되는 페이지가 아닐 수 있습니다.

### 가까운 대체 페이지가 실제로 있는가

있다면 리다이렉트가 자연스러울 수 있습니다.
없다면 억지로 보내도 사용자만 더 헷갈립니다.

예를 들어 구형 상품에 대해 비슷한 후속 모델이 있다면 리다이렉트가 괜찮습니다.

하지만 아주 특정한 구형 모델을 그냥 일반 카테고리로 보내면 관련성이 많이 약해집니다.

---

### 4. URL 오타나 형식 차이 문제라면, 목적지가 확실할 때만 보정하자

404 중에는 페이지 삭제가 아니라 URL 자체의 문제로 발생하는 것도 있습니다.

대표적으로:

- 외부 사이트가 오타가 있는 링크를 걸었다
- 사용자가 주소를 잘못 입력했다
- 대소문자나 마지막 슬래시만 다르다
- 오래된 파라미터가 깨진 URL을 만든다

패턴이 분명하다면 규칙 기반으로 보정해도 됩니다.

예를 들면:

- 대문자 URL을 소문자로 통일하기
- 마지막 슬래시 형식을 맞추기
- 예전 디렉터리 전체를 새 디렉터리로 이동하기

하지만 전제는 하나입니다.

**어디로 보내야 하는지 확실할 때만 해야 합니다.**

404는 수정할 수 있지만, 감으로 보내면 안 됩니다.

---

### 5. 커스텀 404 페이지는 제대로 만드는 편이 좋다

많은 사이트는 404 페이지에 이런 문장 하나만 넣습니다.

> 죄송합니다. 페이지를 찾을 수 없습니다.

기술적으로 틀린 건 아니지만, 사용자 경험 측면에서는 아쉽습니다.

404 페이지에는 최소한 이런 역할이 있어야 합니다.

- 페이지가 존재하지 않는다는 점을 알려 주기
- 홈페이지로 돌아갈 수 있게 하기
- 사이트 검색을 제공하기
- 주요 카테고리로 이동하게 하기
- 다음에 볼 만한 콘텐츠를 안내하기

콘텐츠 사이트라면 저는 이런 요소를 넣는 편입니다.

- 최신 글
- 인기 카테고리
- 사이트 검색

상품 사이트라면 보통 이런 쪽이 더 자연스럽습니다.

- 핵심 카테고리
- 인기 상품
- 문의 또는 피드백 경로

이것만으로 순위가 갑자기 오르는 건 아니지만, 불필요한 이탈은 꽤 줄일 수 있습니다.

## 일반적인 404보다 더 나빠질 수 있는 흔한 처리 실수 4가지

### 1. 죽은 페이지를 전부 홈페이지로 보내기

전형적인 “일단 처리했다” 방식입니다.

에러를 숨길 뿐, 사용자 의도는 해결하지 못합니다.

### 2. 사실상 사라진 페이지가 200을 반환하고 있다

이건 soft 404입니다.

URL은 열리지만 실제 내용은 거의 없는 상태입니다.

사라진 페이지라면 살아 있는 척하지 않는 편이 낫습니다.

### 3. 404를 반환하면서도 내부 링크는 계속 그 페이지를 가리킨다

예를 들면:

- URL은 이미 404다
- 그런데 sitemap 에 남아 있다
- 오래된 글에서 아직 링크하고 있다
- 관련 글 블록에도 계속 노출된다

이러면 신호가 서로 충돌합니다.

### 4. robots.txt 로 404 문제를 가리려 한다

404 처리에서 중요한 건 상태 코드와 링크 정리입니다.

robots.txt 로 숨긴다고 근본 문제가 해결되지는 않습니다.

## 지금 404 페이지 SEO를 점검하려면 이 흐름이면 충분하다

### 1단계: 실제 404 URL 목록을 뽑는다

Search Console, 로그, 크롤러, 분석 도구 등을 활용합니다.

### 2단계: 404를 유형별로 나눈다

최소한 이렇게는 나누는 게 좋습니다.

- 영구 삭제, 대체 없음
- 새 URL로 이동함
- 오타 또는 형식 차이
- 내부 링크 문제
- 외부 링크 문제

### 3단계: 사이트 내부 참조를 정리한다

확인할 곳은:

- 내비게이션
- 본문 링크
- 추천 모듈
- 브레드크럼
- sitemap

### 4단계: 관련성이 강한 경우에만 301을 사용한다

양보다 정확도가 중요합니다.

### 5단계: 404 페이지 자체를 개선한다

거기서 끝나는 화면으로 만들지 않는 게 중요합니다.

## 마무리

404 페이지 SEO에서 진짜 어려운 건 리다이렉트 규칙을 쓰는 일이 아닙니다.

어려운 건 다음 판단을 분리해 내는 일입니다.

- 그 페이지가 정말 끝난 것인가
- 대체 페이지가 있는가
- 사용자를 비슷한 의도의 페이지로 안내해야 하는가
- 사이트가 아직도 새로운 죽은 링크를 내부에서 만들고 있지는 않은가

이 구분이 잘 될수록 사이트 구조는 더 깔끔해집니다.

잘 처리된 404는 기술적인 수치가 아닙니다. 오래된 콘텐츠를 제대로 퇴장시킬 줄 아는 사이트라는 증거입니다.
