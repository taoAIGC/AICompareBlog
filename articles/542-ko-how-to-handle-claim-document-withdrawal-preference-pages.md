## 이미 표시 페이지, 구성 페이지, 설정 페이지, 보충제 철회 옵션 페이지를 정리한 많은 해외 ​​전자상거래 사이트에서 여전히 선호 관련 URL을 일괄 검색 결과에 입력하는 이유는 무엇입니까?

이 문제는 최근에 매우 흔해졌습니다.

많은 팀이 이 지점에 도달하면 실제로 SEO에 대한 개념이 전혀 없습니다. 나는 몇몇 명백한 백엔드 페이지, 프로세스 페이지, 결과 페이지를 무작위로 배치해서는 안 된다는 것을 알고 있습니다. 또한 config, settings, option 등의 URL은 필드가 많기 때문에 기본적으로 순위에 참여할 수 없는 것으로 알고 있습니다.

그러나 시스템이 계속 성장하는 한, 이름이 가볍게 들리고 도움말 페이지로 잘못 판단될 가능성이 더 높은 페이지(기본 설정, 기본 설정, 기본 설정, 사용자 설정, 채널 설정)가 쉬워질 것입니다.

일반 URL은 일반적으로 다음과 같습니다.

- `/claim/document-withdraw-preference`
- `/claim/document-withdraw-preferences`
- `/merchant/document-withdraw/user-preference`
- `/claim/document-withdraw/default-preference`
- `/claim/document-withdraw/channel-preference`
- `/claim/document-withdraw-preference-setting`

많은 사람들이 이러한 유형의 URL이 처음으로 색인화되는 것을 볼 때 직관적으로 몇 가지 판단을 내릴 것입니다.

- 단어 선호도는 백그라운드 작업 페이지처럼 들리지 않으며 공개 설명 페이지로 착각하기 쉽습니다.
- 기본값, 기본 설정 규칙, 디스플레이 구경 등이 페이지에 자주 적혀 있어 빈 페이지처럼 보이지 않습니다.
- 역할과 채널에 따라 선호도가 자주 바뀌는데, 이는 비즈니스에서 매우 중요하다고 생각됩니다.
- 페이지에 필드 설명이 있는 경우가 많아 '이 페이지에는 내용이 있으므로 그대로 유지해도 상관없다'고 오해하기 쉽습니다.
- 시스템이 자동으로 생성했기 때문에 지금은 그냥 놔두세요. 롱테일 단어를 추가해도 될 것 같습니다.

그러나 사이트가 오랫동안 운영되는 한 다음과 같은 현상이 나타납니다. **패치 철회 기본 설정 페이지의 대부분은 본질적으로 시스템의 규칙 기본 설정 페이지, 기본 동작 페이지 및 역할 선택 페이지 유형입니다. "기본적으로 표시하는 방법", "어떤 채널이 어떤 기본 설정을 사용하는지", "어떤 역할이 무엇을 보는지", "우선순위를 지정하는 방법"과 같은 내부 구성 문제를 해결합니다. 이는 검색 트래픽 수신에 적합하다는 공개적이고 안정적이며 장기적인 질문에 대답하지 않습니다. **

검색 사용자가 정말로 알고 싶어하는 것은 일반적으로 다음과 같습니다.

- 패치 철회 기본 설정이 적용되지 않는 이유는 무엇입니까?
- 기본 설정, 설정, 구성의 차이점은 무엇입니까?
- 계정마다 보충제 인출 규칙이 다르게 표시되는 이유는 무엇입니까?
- 기본 환경 설정을 변경한 후에도 프런트엔드가 변경되지 않은 이유는 무엇입니까?
- 사이트, 채널, 역할 간의 우선순위: 누가 누구를 담당하는가?

즉, 검색 트래픽을 받기에 더 적합한 것은 일반적으로 특정 백엔드의 기본 설정 페이지가 아니라 "추가 기능 철회 기본 설정이 적용되지 않는 경우 어떻게 해야 할까요?", "추가 기능 철회 기본 설정 규칙에 대한 설명", "역할 간 기본 설정 차이를 해결하는 방법", "기본 설정과 기본 설정의 차이"와 같은 공개 콘텐츠입니다. **

## 패치 철회 선호 페이지는 어떤 문제를 해결하나요? 우선순위는 처음 검색하는 사람이 아닌 시스템의 운영자에게 서비스를 제공하는 것입니다.

### 1. 패치 철회 과정에서 '기본 방식'을 정의하는 것이 핵심 기능입니다.

대부분의 추가 기능 철회 기본 설정 페이지는 실제로 다음 작업을 수행합니다.

- 다양한 채널, 사이트 및 테넌트에 대한 기본 디스플레이 및 기본 작업 구성
- 다른 캐릭터가 프로세스에 진입할 때 우선순위 적중 규칙을 지정합니다.
- 기본 확인 옵션, 기본 처리 경로 및 기본 알림 전략을 정의합니다.
- 고객 서비스, 운영, 위험 제어 및 기술을 통합하여 동일한 기본 설정에 따라 협업합니다.

직설적으로 말하면 이 유형의 페이지는 사용자 검색을 위해 열리는 답변 페이지가 아니라 시스템 동작에 대한 "기본 전환 지침"에 가깝습니다.

### 2. 일반적으로 계정, 역할, 채널, 버전 및 우선순위와 밀접하게 연관되어 있습니다.

표준 패치 철회 기본 설정 페이지에는 다음 정보가 자주 나타납니다.

-테넌트, 사이트, 채널, 역할 등의 컨텍스트 필드
- 기본 표시 전략, 기본 정렬, 기본 처리기 규칙
- 히트 우선순위, 커버리지 관계, 폴백 전략
- 유효 시간, 그레이스케일 범위, 버전 번호, 업데이트 시간
- 템플릿 바인딩 관계, 알림 기본 설정, 필드 가시성 기본 설정
- 특정 국가 방송국, 특정 캐릭터, 특정 상인에게 별도로 적용되는 조건

이러한 콘텐츠가 구체적일수록 배경 컨텍스트에 더 많이 의존하고 공개 검색 요구에 적합하지 않습니다.

### 3. 비즈니스에서 중요하다고 해서 자연스럽게 포함하기에 적합한 것은 아닙니다.

이는 특히 혼동하기 쉽습니다.

비즈니스 용어로 "중요"는 이 페이지가 프로세스의 원활한 실행에 영향을 미치는지 여부를 나타냅니다. SEO 용어에서 '포함할 가치가 있다'는 것은 해당 URL이 공개적이고 장기적이며 반복적으로 검색되는 질문에 안정적으로 답변할 수 있는지 여부를 나타냅니다.

많은 추가 철회 기본 설정 페이지는 실제로 내부 협업에 매우 중요하지만 이것이 검색 결과에 표시되어야 한다는 의미는 아닙니다.

## 패치 철회 기본 설정 페이지는 어떻게 처리하나요? 먼저 이 5가지 상황을 별도로 살펴보는 것이 좋습니다.

### 1. 표준 환경설정 페이지, 기본 환경설정 페이지, 사용자 환경설정 페이지일 경우에는 일반적으로 핵심 SEO 페이지로 취급할 필요가 없습니다.

가장 일반적인 카테고리입니다.

이러한 유형의 페이지에는 일반적으로 다음과 같은 몇 가지 분명한 특징이 있습니다.

- 특정 계정, 테넌트, 역할, 채널 또는 비즈니스 라인에 바인딩되어야 합니다.
- 페이지의 내용은 규칙을 명백하게 설명하기보다는 "기본 설정이 무엇인지"를 중심으로 전개됩니다.
- 배경 컨텍스트가 없으면 외부 사용자가 이러한 선호도 차이가 존재하는 이유를 이해하기 어렵습니다.
- 페이지의 콘텐츠는 버전, 권한, 채널에 따라 변경되므로 안정적이지 않습니다.
- 본질적으로 공개 콘텐츠 페이지라기보다는 내부 구성 페이지에 가깝습니다.

한마디로: **표준 애드온 철회 기본 설정 페이지는 SEO에 자연스럽게 적합한 랜딩 페이지라기보다는 내부 규칙 입구에 가깝습니다. **

### 2. 실제 검색값이 '왜 해당 환경설정이 효과적이지 않은지'인 경우 개인 환경설정 페이지에서 이를 실행하지 않도록 하세요.

검색이 필요한 팀이 많지 않은데도 잘못된 페이지를 보게 되는 경우가 많습니다.

사용자가 "철회 보완 기본 설정이 적용되지 않음", "기본 기본 설정이 변경되었으나 적용되지 않음" 및 "다른 역할에 따라 표시되는 규칙이 다른 이유"를 검색한다고 해서 특정 `/claim/document-withdraw-preference?id=xxx`가 순위에 적합하다는 의미는 아닙니다.

사용자가 실제로 알고 싶어하는 것은 일반적으로 다음과 같습니다.

- 기본 환경설정이 변경되었으나 적용되지 않는 이유는 무엇입니까?
-어떤 레이어 기본 설정이 더 높은 우선순위를 가지고 있는지
- 역할 선호도가 채널 선호도보다 우선합니까, 아니면 사이트 선호도가 전역 선호도보다 우선합니까?
- 백엔드를 수정한 후 프런트엔드가 동기화되지 않는 이유는 무엇입니까?
- 기본 설정, 설정, 구성, 규칙은 각각 무엇인가요?

이러한 요구 사항은 특정 시스템의 기본 설정 URL보다는 공개 도움말 페이지, FAQ 페이지, 문제 해결 가이드 페이지에 더 적합합니다.

### 3. 공개 도움말 문서이거나 기본 설정 규칙 설명 페이지인 경우 색인을 유지할 가치가 있는지 별도로 평가할 수 있습니다.

기본 설정이 있는 모든 페이지가 모든 페이지에 적합한 것은 아닙니다.

예를 들어, 다음 페이지는 비공개 패치 철회 기본 설정 페이지와 동일하지 않습니다.

- 모든 사용자를 위한 "패치 탈퇴 우선 규칙 설명 페이지"
- "다양한 캐릭터 선호도 우선순위 설명" 페이지 공식 공개
- 특정 계정과 테넌트를 묶지 않는 "기본 환경설정 필드 설명 페이지"
- 검색 사용자를 위한 "기본 설정이 유효하지 않은 문제 해결 가이드" 페이지

이 유형의 페이지가 다음 조건을 충족하는 경우:

- 로그인하지 않고도 이해하실 수 있습니다.
- 해당 내용은 가맹점의 개인 선호 기록이 아닌 공적 규칙입니다.
- 페이지에는 명확한 필드 설명, 우선 순위 관계 및 문제 해결 제안 사항이 있습니다.
- URL은 안정적이며 생존을 위해 임시 매개변수나 세션 상태에 의존하지 않습니다.
- 명확한 검색 요구 사항에 실제로 연결

그러면 이는 공개 도움말 페이지와 비슷하며 포함 여부를 독립적으로 평가할 수 있습니다.

### 4. 시스템이 선호도, 선호도, 기본 선호도, 사용자 선호도, 채널 선호도, 선호도 로그 등의 변형 URL을 동시에 생성하는 경우 반드시 함께 제어하십시오.

많은 사이트의 실제 문제는 단순한 `/claim/document-withdraw-preference`가 아닙니다.

보다 일반적으로 시스템은 일련의 변형도 확장합니다.

- `/claim/document-withdraw-preference`
- `/claim/document-withdraw-preferences`
- `/claim/document-withdraw-default-preference`
- `/claim/document-withdraw-user-preference`
- `/claim/document-withdraw-channel-preference`
- `/claim/document-withdraw-preference-log`

이러한 URL이 많으면 쉽게 표시됩니다.

- 페이지의 본문은 유사하지만 개체와 매개변수가 다릅니다.
- 기본 환경설정 페이지, 사용자 환경설정 페이지, 로그 페이지가 함께 캡처됩니다.
- 도움말 센터, 메시지 알림 및 백그라운드 탐색을 통해 이러한 페이지에 대한 입구가 지속적으로 제공됩니다.
- 검색 엔진은 어떤 URL이 실제로 보관되어야 하는 공개 페이지인지 알 수 없습니다.

따라서 추가 철회 기본 설정 페이지를 처리할 때 매우 실용적인 조치는 다음과 같습니다.

**기본 설정 페이지 자체에만 집중하지 말고 기본, 사용자, 채널, 로그 문자열을 함께 정렬하세요. **

### 5. 패치 철회 선호 페이지를 순위에 참여시키지 않으려면 NOINDEX, 로그인 차단, 사이트맵, Canonical, 매개변수 제어를 함께 통합합니다.

많은 부가 기능이 선호 페이지 SEO 문제를 철회하는 이유는 "이 페이지가 있기 때문"이 아니라 신호가 서로 싸우기 때문입니다.

일반적인 상황은 다음과 같습니다.

- 페이지에 색인이 없지만 사이트맵이 계속해서 기본 설정 URL을 제출합니다.
- 기본 기본 설정 페이지에는 로그인이 필요하지만 매개변수가 있는 사용자 기본 설정 페이지에는 익명으로 접근할 수도 있습니다.
- 혼돈에 대한 정식 포인트, 기본 선호도 및 채널 선호도 페이지는 서로의 신호를 놓고 경쟁합니다.
- 테넌트, 사이트, 채널, 역할 및 버전 매개변수가 포함된 다양한 URL 변형이 캡처되었습니다.
- 실제로 검색 가능한 설명 페이지가 너무 얇습니다. 반대로 개인 기본 설정 페이지는 필드와 설명이 많기 때문에 콘텐츠 페이지에 더 가깝습니다.

추가 철회 기본 설정 페이지를 SEO 입구로 사용해서는 안 된다고 결정했다면 절반만 변경하지 마세요. 관련 신호를 한 번에 통합하는 것이 가장 좋습니다.

## 이제 추가 기능 및 기본 설정 페이지를 철회할 때 발생하는 가장 일반적인 4가지 SEO 실수를 확인했습니다. 그렇게 하면 사이트를 더 쉽게 엉망으로 만들 수 있습니다.

### 1. '선호 규칙이 많다'는 것은 '포함하기에 적합하다'는 뜻이라고 생각합니다.

규칙이 많다고 해서 검색값이 높다는 의미는 아닙니다. 많은 기본 설정 페이지가 풍부해 보이지만 이러한 내용은 현재 시스템, 현재 역할 및 현재 버전에만 유효합니다.

### 2. 기본 설정 페이지만 제어하고 기본, 사용자, 채널, 로그 및 기타 변형 페이지는 제어하지 않습니다.

표면적으로는 기본 URL을 관리한 것처럼 보이지만 실제로는 가치가 낮은 페이지가 계속해서 크롤링 예산을 소모합니다.

### 3. 당연히 "기본 설정이 적용되지 않는 경우 어떻게 해야 할까요?"에 대한 공개 콘텐츠를 만들어야 하지만, 우리는 항상 개인 기본 설정 페이지가 트래픽을 받기를 원합니다.

실제 장기적인 가치는 일반적으로 특정 백엔드의 기본 설정 페이지가 아닌 공개 설명 페이지와 문제 해결 페이지입니다.

### 4. 백그라운드 룰은 변경되었으나, 인덱스 전략은 함께 업데이트되지 않았습니다.

많은 팀이 기본 설정 우선순위, 역할 수준, 기본값 및 표시 논리를 변경하지만 사이트맵, 표준, 매개변수 전략 및 권한 확인은 그 어느 하나도 따라가지 못하며 최종 색인은 여전히 ​​엉망입니다.

## 지금 홈페이지에서 패치철회 선호페이지를 확인하고 싶으시다면 이 순서대로 진행하시는 것을 추천드립니다.

### 1단계: 먼저 모든 환경설정 관련 URL을 알아보세요.

최소한 다음 유형을 제거하십시오.

- 출금 선호 페이지 보완
- 추가 기능은 기본 환경설정 페이지를 불러옵니다.
- 탈퇴 사용자 기본 설정 페이지 보완

- 패치 출금 선호 로그 페이지
- 테넌트, 사이트, 채널, 역할, 버전 매개변수가 포함된 기본 설정 URL

### 2단계: 어떤 질문이 검색에 실제로 가치가 있는지 결정

사용자가 실제로 다음을 검색하고 있는지에 집중하세요.

- 보충제 철회 우선권이 적용되지 않는 이유는 무엇입니까?
-기본 설정, 설정, 구성의 차이점은 무엇입니까?
- 계정마다 규칙이 다른 이유는 무엇입니까?
- 기본 환경설정이 변경된 후에도 프런트엔드가 변경되지 않은 이유는 무엇입니까?
- 선호도 우선순위가 충돌할 때 가장 먼저 확인해야 할 레이어는 무엇인가?

이러한 질문이 검색에 꼭 필요한 경우 개인 기본 설정 페이지를 계속해서 맨 위로 이동시키는 대신 공개 설명 페이지를 별도로 만드십시오.

### 3단계: 공개 도움말 페이지와 내부 기본 설정 페이지를 완전히 분리합니다.

다음을 명확하게 구분해야 합니다.

- 검색 사용자를 위한 공개 설명 페이지는 어떤 페이지입니까?
- 운영, 고객 서비스, 기술을 위한 내부 전용 페이지는 무엇입니까?
- 기록 페이지, 로그 페이지, 기본값 페이지는 어떤 페이지인가요?

이러한 유형의 페이지를 동일한 색인 전략에 혼합하지 마십시오. 그렇지 않으면 변경 사항으로 인해 페이지가 더욱 혼란스러워질 것입니다.

### 4단계: 크롤링 및 색인 생성 신호의 통합 처리

추가 철회 선호 페이지가 순위에 참여하지 않아야 하는 경우 다음 신호를 즉시 처리하십시오.

- NOINDEX 규칙
- 로그인 차단 또는 권한 확인
- 사이트맵 제출 전략
- 정식 포인트
- 매개변수 URL 제어
- 사이트 진입 및 내부링크 전략
- 프런트엔드 출력 및 서버사이드 렌더링 전략

오늘 기본 기본 설정 페이지를 변경하지 말고 내일 기본 기본 설정 페이지, 사용자 기본 설정 페이지 및 로그 페이지에서 동일한 URL 배치를 검색 엔진에 계속 공급하세요.

### 5단계: 결과를 볼 때 '기본 설정 페이지가 삭제되었나요?'에만 집중하지 마세요.

- 사용자가 "기본 설정이 적용되지 않는 이유는 무엇입니까?"라고 검색하면 실제로 표시하려는 공개 페이지가 아닌 내용이 표시됩니다.
