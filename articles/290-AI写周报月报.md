## 具体操作方法

### 第一步：整理本周/本月工作内容

在写周报月报之前，先把自己这周/月做的事情简单列出来：

**可以这样整理**：
- 完成了哪些项目
- 参与了哪些会议
- 遇到了什么问题（ Optional ）
- 下周/月计划做什么

**提示**：最好每天简单记一下要不然到写周报的时候想不起来具体做了什么。

### 第二步：选择合适的AI工具

| 工具 | 特点 | 适合场景 |
|------|------|----------|
| Kimi | 长文本处理强 | 内容多的详细汇报 |
| 通义千问 | 稳定可靠 | 常规工作汇报 |
| 文心一言 | 联网查数据 | 需要补充数据的汇报 |
| 讯飞星火 | 语音输入 | 喜欢用语音的人 |

### 第三步：给出清晰的指令

这是最关键的一步！指令给的好，AI才能写得好。

**万能公式**：
```
你是一个[职业]，帮我写一份[周报/月报]，内容包括：[具体内容]，要求：[格式/长度/语气]
```

**具体例子**：

**场景1：写周报**
```
你是一个产品经理，帮我写一份本周工作周报，包括：1.完成的需求评审 2.和开发对接的情况 3.用户反馈处理 4.下周计划。要求：条理清晰，每点不超过50字，语正式。
```

**场景2：写月报**
```
你是一个市场专员，帮我写一份月度工作总结，包括：1.本月完成的主要工作 2.取得的成绩（最好有数据） 3.存在的问题 4.下月计划。要求：用数据说话，语言专业正式，800字左右。
```

### 第四步：检查和修改

AI写完后，一定要检查以下几点：

1. **事实核对**：确认AI写的项目确实是你做的
2. **数据核实**：如果有数据，一定要核实准确
3. **语气调整**：根据自己和领导的习惯调整语气
4. **隐私检查**：注意不要泄露敏感信息

## 常见问题

### Q1：AI写出来的周报太笼统怎么办？
A：提供更具体的工作内容。你给的细节越多，AI写得越具体。

### Q2：领导一眼就能看出是AI写的吗？
A：只要经过检查和适当修改，一般看不出来。关键是要加入你自己的实际工作内容。

### Q3：可以用AI直接生成假的工作内容吗？
A：千万不能！周报月报要实事求是，AI只是帮你整理和表达，不是帮你编造内容。

### Q4：不同公司对周报月报要求不同怎么办？
A：可以在指令中明确告诉AI你的公司要求，比如"我们公司周报要求简洁，每点不超过3行"。

## 进阶技巧

### 1. 建立自己的模板
每次用AI写完后，保存一个你自己习惯用的模板，下次直接套用。

### 2. 积累常用表达
把AI写得好的一些表达记下来，下次可以直接用。

### 3. 结合语音输入
如果内容很多，可以先用语音说，AI帮你整理成文字，再优化。

### 4. 让AI帮你想问题
可以问AI"作为一个产品经理，周报通常应该包含哪些内容"，让它帮你查漏补缺。
