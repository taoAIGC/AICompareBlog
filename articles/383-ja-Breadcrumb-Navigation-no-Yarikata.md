## 多くの Web サイトには明らかにブレッドクラム ナビゲーションがあるのに、SEO の意味がほとんどないのはなぜですか?

私はこれが起こるのを何度も見てきました。

多くの人は非常に単純なアイデアでパンくずリストのナビゲーションを行っています。

- ページの上部に「ホーム > カテゴリ > 記事」の行を配置します。
- 視覚的にはそのように見えます
- ユーザーはクリックすることもできる場合があります
- デフォルトでは、パンくずリストのナビゲーションはすでに完了しています。

しかし、SEO に関しては、あまり重要な役割を果たしていないことがよくあります。

理由は複雑ではありません。**多くの Web サイトは「ブレッドクラムのように見え」、「検索エンジンが構造を理解するのに実際には役立っていません」。 **

たとえば、よくある質問には次のようなものがあります。

- 分類レベルが乱雑です。今日はここに掛けて、明日はあそこに掛けます。
- 現在のページが複数の異なるパスに同時に表示されます
- ページにはブレッドクラムがありますが、コードにはブレッドクラム スキーマがありません
- 携帯電話上に直接隠されているため、ユーザーも検索エンジンもロジックを明確に見ることができません
・記事ページのみにあり、コラムページ、商品ページにはありません。

最近ブレッドクラム ナビゲーションの作り方を勉強している方に、非常に簡単な結論をお伝えします。

**ブレッドクラム ナビゲーションは、「ページをよりプロフェッショナルに見せる」ためではなく、Web サイトの階層、ユーザー パス、内部リンクの関係を明確に説明するためのものです。 **

これが正しく行われると、ユーザーはよりスムーズにページを見つけられるようになり、検索エンジンはウェブサイトの構造をより安定して理解できるようになります。

## ブレッドクラム ナビゲーションは具体的に何に役立つのでしょうか?単なる「戻るボタン」と考えないでください

パンくずリスト ナビゲーションに対する多くの人の理解は、依然として「ユーザーが前のレベルに戻るのに便利」という程度に限られています。

確かにその通りですが、本当に価値があるのはここだけではありません。

Web サイトのブレッドクラムを見るときは、次の 3 つのことがうまく行われているかどうかに焦点を当てます。

### 1. ユーザーに、現在どのレイヤーにいるのかを伝えます。

特に、コンテンツが多くカテゴリーが深い Web サイトの場合、ユーザーは毎回ホームページからアクセスするわけではありません。

多くの人は、検索結果から特定の記事、製品ページ、トピック ページに直接アクセスします。

このとき、ブレッドクラム ナビゲーションの役割は、次のことを一目で知らせることです。

- このページはどの列に属しますか?
- 次のレベルには他に何がありますか?
- 次にどこに続ければよいですか?

### 2. 検索エンジンに伝えます: Web サイト構造内でこのページの所有者は誰ですか?

これはSEOにとってさらに重要です。

明確なブレッドクラム パスは実際に検索エンジンに次のことを伝えます。

> このページは独立したコンテンツではなく、Web サイト全体において明確な所有権を持っています。

Web サイトの階層が明確であればあるほど、検索エンジンが理解しやすくなります。

- 上の列は何ですか?
- 具体的なコンテンツページとは何ですか?
- どのページに親子関係がありますか?

### 3. 自然な内部リンク関係を作成する

パンくずリストは基本的に内部リンクです。

しかも、「リンクを追加するためにリンクを追加する」というハードなものではなく、本質的に合理的でユーザーがクリックしたくなるパス型リンクです。

適切に実行されたブレッドクラム ナビゲーションは、ユーザーが前のレベルに戻るのに役立つだけでなく、より安定した内部リンクのサポートを備えたいくつかのコア列ページを提供します。

## ブレッドクラム ナビゲーションを使用するにはどうすればよいですか?むしろ、最初にこれら 5 つの詳細を正しく理解することをお勧めします。

### 1. まず Web サイトのレベルを決定し、次にパンくずリストを記述します。そのまま書き込まないでください。

多くの Web サイトのパンくずリストの最大の問題は、スタイルが見栄えが悪いことではなく、アクセス パス自体が不安定であることです。 **

たとえば、同じタイプの記事:

- 今日は「SEO チュートリアル」にぶら下がっています。
- 明日には「コンテンツマーケティング」に掲載されます
- 明後日、キーワードを集めるために「ウェブマスターアカデミー」のレイヤーを追加します

この結果、ブレッドクラムは混雑しているように見えますが、実際にはこのページがどこに属しているのか誰も理解できません。

したがって、私の習慣は常にシンプルです。

**最初に Web サイトのレベルを決定し、次にブレッドクラムにこのレベルを正確に反映させます。 **

たとえば、記事が長期にわたる場合、その記事は次のものに属します。

ホーム > SEO 最適化 > テクニカル SEO > 記事の詳細

それから、今日はあることに変えて、明日は別のことに変えるのはやめましょう。

安定したパスは派手なパスよりも重要です。

---

### 2. 前のレベルをクリックできる必要があります。また、現在のページで自分自身にリンクしないでください。

実用的ですが見落とされがちな細かい点:

**パンくずリストの実際の値は、現在の最後のページではなく、前の階層リンクです。 **

したがって、私は通常次のように処理します。

- ホームページはクリック可能です
- カテゴリページをクリック可能
-クリック可能なサブカテゴリーページ
- 現在のページが表示されるだけで、自己リンクは追加されません。

理由は簡単です。

ユーザーがブレッドクラムをクリックするときは、通常、現在のページを更新するのではなく、上に移動したいと考えます。

最後の項目もクリック可能にしてしまうと、体験的には何のメリットもありませんし、構造的にも情報量は増えません。

無視できないことがもう 1 つあります。

**リッチに見せるためだけにレイヤーを詰め込みすぎないでください。 **

パスが長すぎてユーザーが最後までやりたくない場合、そのナビゲーションの価値はすでに低下しています。

---

### 3. パンくずリストのパスは、URL、タイトル、正規表現とできる限り一致させる必要があります。

このステップは多くの人にとって最も簡単です。

表面上はブレッドクラム ナビゲーションがあるページをいくつか見たことがありますが、バックグラウンド シグナルが競合しています。

- ブレッドクラムには、この記事がカテゴリー A に属することが示されています
- URL はカテゴリ B です
- ページタイトルは C テーマを強調しています
- canonical は別のバージョンを指します

この場合、検索エンジンが受け取るのは「明確な構造」ではなく、「自分で考えてもいないもの」です。

したがって、より安定したアプローチは次のとおりです。

- ページには可能な限りメインの属性を 1 つ含める必要があります
- パンくずリストのパスは、ページの実際のレベルと可能な限り一致している必要があります。
- URL、コラム、カノニカルはお互いについて話さない

特にコンテンツ サイトは、「より多くの単語を占めるために、記事が同時に複数のパスに分類されること」を最も恐れています。

短期的には、より多くの道路を舗装するように見えますが、長期的には、構造を台無しにする方が簡単です。

---

### 4. パンくずリストが必要なのは記事ページだけではなく、深いコラムページや商品ページにも必要です。

多くの Web サイトでは、非常に典型的な問題が発生します。

**パンくずリストは記事の詳細ページにのみ配置され、その他の詳細なページはユーザー自身の推測に任されています。 **

ただし、ユーザー パスと Web サイト構造の最適化の観点から見ると、実際には次のページを追加するのが非常に適しています。

- 二次コラムページ
- レベル 3 トピック ページ
- 商品詳細ページ
- ヘルプセンターの詳細ページ
- ドキュメントページ

特に、多くの商品、深いカテゴリ、大量のコンテンツがあるサイトでは、ユーザーは先に進むほど迷ってしまう可能性が高くなります。

現時点では、ブレッドクラム ナビゲーションは必要不可欠な装飾ではなく、ユーザーがブラウジングを継続できるようにするための重要な手段です。

もう一つの経験は次のとおりです。

**面倒な場合は、モバイル上で非表示にするだけではだめです。 **

スタイルを簡素化することはできますが、完全に削除しないでください。

実際のトラフィックの多くはモバイル側から来ています。モバイル側に明確なブレッドクラム ナビゲーションがない場合、ユーザーが前のレベルに戻ることがより困難になり、構造上のシグナルが 1 つ少なくなります。

---

### 5. ページ上に表示するだけでは不十分です。ブレッドクラム スキーマを追加するのが最善です。

これはテクニカル SEO において非常に重要なステップです。

パンくずリスト ナビゲーションを作成したら、多くの人は作業が終わったと考えます。

実際には、まだ 1 つのステップが残っています。**このリンク文字列がブレッドクラム パスであることを検索エンジンに通知します。 **

最も一般的な方法は、「BreadcrumbList」のような構造化データを追加することです。

その意味は次のとおりです。

- 検索エンジンが階層関係をより明確に理解できるようにします
- パンくずリスト信号が正しく認識される確率を向上させます。
- 検索結果でより明確なパスを表示する機会

したがって、フロントエンド表示のみを行っており、ブレッドクラム スキーマをまだ行っていない場合は、それを作成することをお勧めします。

完了した後は急いで完了させないでください。もう一度確認することをお勧めします。

- 階層順序はページ表示と一致していますか?
・リンク先のアドレスは正式なURLですか？
- 現在のページ位置を正しくマークしましたか?
- テスト ツールで報告された構造上のエラーはありますか?

「ページ上にあるように見える」が「実際の検索エンジンが認識しない」にならないようにしてください。

## 通常、ブレッドクラムを強制しないページはどれですか?

すべてのページがこれを行う必要があるわけではありません。

一般的に推奨しないシナリオには次のようなものがあります。

- 単一コンバージョンのランディング ページ
- 非常に軽く構成された 1 ページのページ
- 臨時イベントページ
- 目的が 1 つだけであり、閲覧を続ける必要がほとんどないページ

このタイプのページは、存続するために階層ブラウズに依存しないためです。

ただし、ページが比較的明確な Web サイト構造、特にコンテンツ ページ、製品ページ、ドキュメント ページである限り、基本的には本格的なバージョンを作成することをお勧めします。

## パンくずリストを追加してもSEOが改善されない場合は、まず次の4点を確認してください。

### 1. パスは「正しく見える」だけで、実際には統一されていませんか?

同じタイプのページのパスが変化し続ける場合、パンくずリストがどれほど多くても、それは単なるパフォーマンス ナビゲーションになります。

### 2. このページは本当に Web サイトの構造に引っかかっているのでしょうか?

ページ自体が孤立した島のようなもので、列の入り口もテキスト内に内部リンクもない場合、単一のブレッドクラムでは全体の構造シグナルを駆動することはできません。

### 3. スキーマが欠落しているか、間違って記述されていますか?

フロントエンドの表示が正常だからといって、検索エンジンがそれを正しく理解する必要があるわけではありません。

### 4. 上の列は強化する価値がありますか?

カテゴリ ページ自体のコンテンツが非常に薄く、タイトルが乱雑で、継続的に更新されていない場合、パンくずリストが重みとパスを後退させることになり、その効果は特に理想的ではありません。

したがって、パンくずリスト自体に注目するだけでなく、上部の列ページが維持されているかどうかを振り返って確認してください。

## 最後に言いたいこと

パンくずナビゲーションを作成するにはどうすればよいですか?本当に複雑なことではありません。

最も恐れているのは、そのリンクの行の書き方がわからないことではなく、それを純粋なスタイル コンポーネントとして扱い、書き終わったら忘れてしまうことです。

本当に便利なブレッドクラム ナビゲーションでは、次の 3 つのことを同時に行う必要があります。

- ユーザーは自分がどこにいるかを一目で知ることができます
- 検索エンジンはページの所有権を一目で理解できます
- ウェブサイトの内部階層と内部リンクの関係がよりスムーズになります

最近 Web サイトを整理している場合は、ブレッドクラム ナビゲーションを取り出してチェックしてみるとよいでしょう。

多くの Web サイトには十分なコンテンツがありませんが、構造とシグナルが乱雑すぎます。

ブレッドクラムは、目立たないように見えても後で追加する価値のある詳細であることがよくあります。
