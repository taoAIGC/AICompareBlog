## 为什么产品经理需要多AI对比？

### 场景一：写需求文档

假设你要写一个"用户收藏功能"的需求文档。

同样的问题，不同AI的视角完全不同：

**ChatGPT**（结构化思维）：
```
## 用户故事
- 作为用户，我希望收藏喜欢的商品，以便以后快速找到
## 功能描述
- 收藏入口：商品详情页、购物车、搜索结果页
## 验收标准
- 点击收藏后，图标变为已收藏状态
```

**Claude**（用户视角）：
```
## 用户痛点
- 用户遇到喜欢的商品，但当时不打算购买，事后找不到
## 解决方案
- 提供一键收藏功能，降低用户决策成本
## 交互设计
- 收藏后有即时反馈，如 toast 提示或图标变化
```

**Kimi**（中文优化）：
```
## 需求背景
姐妹们！有没有遇到过这种情况：看到喜欢的衣服，当时没买，
回头怎么也找不到链接了？我们的收藏功能就是来解决这个痛点的！
## 核心功能
- 一键收藏：点一下就行，不用跳转
- 快速找到：我的收藏里一目了然
- 降价提醒：收藏的商品降价了，第一时间通知你
```

三个AI的回答各有侧重：
- ChatGPT 给了标准结构
- Claude 补充了用户痛点分析
- Kimi 让表达更接地气

**融合起来，就是一份完美的需求文档。**

### 场景二：竞品分析

分析竞品App的一个功能设计：

**Perplexity**（搜索最新信息）：
> "根据用户评价，竞品A的收藏功能上线3个月，
> 收藏转化率提升了15%。用户普遍反馈'找商品更方便了'。"

**ChatGPT**（总结分析）：
> "竞品A的收藏功能特点：1. 一键收藏 2. 分类管理 
> 3. 降价提醒 4. 社交分享。优点是交互流畅，
> 缺点是收藏入口较深。"

**Claude**（建议参考）：
> "从产品设计角度，建议：1. 收藏入口放在更显眼的位置 
> 2. 增加收藏夹分类功能 3. 提供批量操作。"

### 场景三：用户访谈提纲

**ChatGPT**（标准模板）：
```
1. 您平时有收藏商品的习惯吗？
2. 您通常在什么情况下会使用收藏功能？
3. 您对现有竞品的收藏功能满意吗？
4. 您希望收藏功能增加什么新特性？
```

**Claude**（深度问题）：
```
1. 能否描述一次您想收藏但最终没有收藏的经历？
2. 是什么阻止您使用收藏功能？
3. 如果收藏的商品降价了，您希望以什么方式收到通知？
4. 您会定期查看自己的收藏夹吗？
```

**Kimi**（口语化）：
```
1. 姐妹们！大家平时买东西会收藏吗？为啥呀？
2. 有没有明明收藏了，结果再也找不到的情况？
3. 你们希望收藏夹能增加啥功能？
4. 收到降价提醒会买吗？啥时候买？
```

## 核心功能详解

### 1. 多站对比模式

一个页面同时展示多个AI的回答：

```
┌─────────────────────────────────────────────────────────┐
│  🤖 ChatGPT         🤖 Claude         🤖 Kimi           │
├─────────────────────────────────────────────────────────┤
│  "我认为这个功能     "从技术实现角度，    "这个需求的核心     │
│   的核心是用户体     我建议分三期实      是解决用户痛       │
│   验的优化……"       现……"              点……"             │
└─────────────────────────────────────────────────────────┘
```

**对比方式**：
- 横向对比：同一行看不同AI的回答
- 竖向阅读：每个AI的回答是完整的
- 快速复制：每栏有独立的复制按钮

### 2. 快捷键唤出

按 `Ctrl+M`（Windows）或 `Cmd+M`（Mac），立即弹出输入框。

不需要点击图标，不需要找插件——**键盘侠的福音**。

### 3. 划词查询

浏览竞品App时，看不懂某个功能设计？

**选中文字** → 旁边出现AI图标 → 点击 → 多个AI帮你分析。

比如选中"收藏转化率"，Claude会解释这是什么意思，ChatGPT会告诉你怎么计算，Perplexity会搜索行业数据。

### 4. 标签页模式

对于高频用户，开启「标签页模式」后，每次新建标签页都会自动打开多AI对比视图。

**适合场景**：一边看竞品，一边让AI分析。

### 5. 自定义AI组合

你可以在设置里预设常用的AI组合：

- **产品工作流**：ChatGPT + Claude + Kimi
- **竞品分析流**：Perplexity + ChatGPT + Claude
- **文案工作流**：Kimi + 豆包 + ChatGPT

一键切换，不需要每次都勾选。
