## Por que tanta gente acha que sitemap bom é sitemap cheio?

Isso acontece direto.

O site entra no ar, a pessoa começa a olhar SEO com mais seriedade e faz o seguinte:

- gera o sitemap
- coloca todas as URLs possíveis
- envia no Search Console
- fica esperando indexação e ranking

Parece capricho, mas nem sempre ajuda.

Na prática, **sitemap bom não é o maior. É o mais limpo.**

Se você coloca páginas de teste, URLs com parâmetros, páginas duplicadas, páginas fracas e páginas com noindex, o buscador não entende “site organizado”. Ele entende “nem o dono decidiu o que realmente importa”.

Então, se você está pesquisando como enviar um sitemap, fica a ideia principal:

**O trabalho não é mandar mais URLs. É mandar as URLs certas.**

---

## O que o sitemap realmente faz? Não trate como botão de indexação automática

O sitemap é, no fundo, uma lista.

Você está dizendo ao buscador:

> Estas são as URLs oficiais do meu site e valem ser verificadas primeiro.

Isso não significa “indexe agora”.

Ele ajuda a:

1. descobrir páginas mais rápido
2. entender qual versão da URL é a principal
3. organizar melhor a estrutura do próprio site

Mas não substitui conteúdo bom, links internos nem valor real da página.

## Como enviar um sitemap? Eu prefiro acertar estes 5 pontos primeiro

### 1. Envie apenas URLs oficiais que você realmente quer indexar

Minha regra é simples:

**Se esta página fosse indexada hoje, eu ficaria satisfeito com isso?**

Se a resposta for não, ela não entra no sitemap.

Normalmente eu deixo de fora:

- URLs com parâmetros de rastreamento
- páginas de busca interna
- páginas de teste ou preview
- filtros e tags claramente duplicados
- URLs com canonical para outra página

### 2. Em site novo, mande primeiro as páginas principais

Muitos sites novos não têm problema de sitemap. Têm problema de página fraca demais logo no começo.

Eu começaria por:

- home
- categorias principais
- artigos mais importantes
- páginas completas e bem estruturadas

### 3. Não falsifique o lastmod

Tem sistema que atualiza o lastmod de tudo todo dia, mesmo sem mudança real. Isso gera ruído.

O ideal é:

- atualizar lastmod só quando houve mudança real no conteúdo
- não mudar por ajustes mínimos
- não dar o mesmo horário para todas as URLs

### 4. Se o site cresceu, divida o sitemap por tipo

Eu prefiro separar por:

- artigos
- produtos
- categorias
- idiomas

Fica muito mais fácil encontrar o problema depois.

### 5. Depois de enviar, acompanhe o retorno

Enviar é só metade do trabalho.

Eu costumo olhar:

- se a quantidade enviada faz sentido
- se “descoberta, mas não indexada” está acumulando
- se muitas URLs estão sendo excluídas
- se há conflito entre sitemap, canonical, noindex e robots

---

## Que páginas eu normalmente deixo fora do sitemap?

Eu quase sempre excluo:

- páginas 404 ou 410
- URLs redirecionadas
- páginas com noindex
- URLs canonicalizadas para outra página
- duplicatas com parâmetros
- busca interna, login, carrinho e páginas funcionais
- páginas incompletas

Regra rápida:

**Não entregue ao Google páginas que você mesmo não quer destacar.**

---

## Se o sitemap foi enviado e a indexação continua lenta, revise estes 4 pontos

### 1. A página realmente merece ser indexada?

### 2. Ela tem entrada interna?

Sem links internos, muita página fica isolada.

### 3. Os sinais técnicos estão brigando entre si?

Exemplo clássico:

- o sitemap envia uma URL
- o canonical aponta para outra
- robots bloqueia parte do caminho
- a página ainda tem noindex

### 4. O rastreamento ou o servidor estão instáveis?

Se o site está lento ou com erro, reenviar sitemap não vai resolver.

---

## Para fechar

Enviar um sitemap não é difícil.

O erro mais comum não é não saber gerar o arquivo. É usar o sitemap como botão mágico quando a indexação anda devagar.

Meu processo é simples:

- limpar as URLs primeiro
- enviar só as páginas oficiais
- observar o retorno
- se algo estiver errado, voltar para a qualidade da página e para a estrutura interna

É aí que o sitemap realmente começa a ajudar.
