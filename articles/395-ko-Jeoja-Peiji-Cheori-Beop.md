## 왜 많은 사이트는 다중 작성자 기능을 켜자마자 트래픽보다 빈 작성자 페이지부터 늘어날까?

이 문제는 블로그, 미디어 사이트, 리뷰 사이트, SaaS 도움말 센터에서 아주 흔합니다.

CMS 기능을 켜면 이런 페이지가 자동으로 생깁니다.

- `/author/admin`
- `/author/editor-01`
- `/author/guest-post`
- `/author/marketing-team`

처음에는 좋아 보입니다.

- 사이트가 더 전문적으로 보이고
- 작성자 정보도 보이고
- E-E-A-T에도 좋을 것 같고
- 작성자 페이지마다 검색 유입을 얻을 수도 있을 것 같으니까요

그런데 조금 지나서 보면 많은 페이지가 이렇습니다.

- 이름과 아바타만 있음
- 아래에는 글 목록만 있음
- 글 1개만 쓴 작성자도 있음
- 실제 공개용 인물이 아니라 내부 계정도 섞여 있음

이때 운영팀은 보통 두 극단으로 갑니다.

- “작성자 정보는 SEO에 좋다”면서 전부 열어두거나
- “얇은 아카이브일 뿐”이라며 전부 막아버리거나

둘 다 너무 거친 처리입니다.

**작성자 페이지는 자동으로 인덱스해야 하는 것도 아니고, 자동으로 막아야 하는 것도 아닙니다. 중요한 건 이 페이지가 실제 작성자, 분명한 전문성, 충분한 콘텐츠, 독립적인 가치를 갖고 있느냐입니다.**

## 작성자 페이지는 어떤 문제를 해결할까?

단순히 사람별 글 목록을 만드는 것만이 아닙니다.

### 1. 누가 글을 썼는지 보여준다

특히 다음 같은 분야에서는 중요합니다.

- SEO
- 기술
- 금융
- 건강
- 법률
- B2B

### 2. 한 사람의 전문성을 하나의 입구로 묶어준다

한 작성자가 특정 주제를 꾸준히 쓴다면, 작성자 페이지는 다음을 이해하기 쉽게 해줍니다.

- 이 사람이 강한 분야
- 주로 다루는 주제
- 어디서부터 읽으면 좋은지

### 3. 사이트의 신뢰도를 높여준다

좋은 콘텐츠라도 누가 썼는지 안 보이면 익명처럼 느껴집니다.

괜찮은 작성자 페이지는 이 약점을 보완합니다.

## 작성자 페이지는 어떻게 처리할까? 먼저 5가지 경우로 나눠서 보는 게 좋다.

### 1. 꾸준히 쓰고 전문 분야가 분명한 핵심 작성자는 정식 페이지로 유지해도 된다

이 사람이:

- 꾸준히 글을 쓰고
- 주제 축이 분명하고
- 글 수가 어느 정도 쌓였고
- 브랜드 신뢰에도 도움이 된다면

작성자 페이지는 충분히 중요한 자산이 될 수 있습니다.

이런 정보가 있으면 더 좋습니다.

- 제대로 된 소개
- 전문 분야
- 대표 글
- 왜 이 작성자를 봐야 하는지에 대한 설명

### 2. 1회성 기고자, 게스트 작성자, 내부 계정은 굳이 인덱스할 필요가 없을 수도 있다

여기서 많은 사이트가 과하게 넓힙니다.

CMS에 계정이 있다고 해서 모두 공개 페이지가 필요한 건 아닙니다.

예를 들면:

- 글 1개 쓴 인턴
- 한 번만 기고한 외부 필자
- admin, editor 같은 공용 계정
- 팀 공용 발행 계정

이런 페이지는 대체로 너무 얇습니다.

### 3. 아바타와 글 목록만 있는 페이지에 큰 SEO 가치를 기대하기는 어렵다

이건 정말 흔한 문제입니다.

페이지는 최소한 다음 질문에 답해야 합니다.

- 이 사람은 누구인가
- 왜 믿을 수 있는가
- 주로 무엇을 쓰는가
- 어떤 글부터 읽으면 좋은가

그래서 가능하면 다음을 넣는 편이 좋습니다.

- 명확한 소개문
- 전문 주제
- 추천 글
- 필요하다면 경력이나 자격 정보

### 4. 작성자 페이지, 소개 페이지, 팀 페이지, 전문가 페이지는 역할을 나눠야 한다

여러 템플릿이 같은 이야기를 반복하는 사이트가 많습니다.

정리하면 이렇게 나누는 편이 깔끔합니다.

- **작성자 페이지**: 이 사람이 무엇을 쓰고 어디서부터 읽으면 되는가
- **소개 페이지**: 회사나 사이트는 누구인가
- **팀 페이지**: 누가 운영하는가
- **전문가 페이지**: 자격, 경력, 전문적 배경

### 5. 중요하지 않은 작성자 페이지는 애매하게 열어두지 말고 신호를 맞춰라

중요하지 않다고 하면서 동시에:

- 전부 sitemap에 넣고
- 템플릿 여기저기서 링크하고
- 내용은 거의 비워두면

결국 약한 페이지가 늘어날 뿐입니다.

핵심 작성자 페이지와 그렇지 않은 페이지를 나누고, sitemap·내부링크·인덱싱 정책을 같은 방향으로 맞추는 게 낫습니다.

## 작성자 페이지에서 자주 보는 SEO 실수 4가지

### 1. CMS 사용자 전원에게 공개 페이지를 만든다

내부 계정과 공개 가치가 있는 작성자는 다릅니다.

### 2. 작성자가 누구인지 설명하지 않으면서 순위만 기대한다

맥락과 신뢰가 없으면 약한 페이지입니다.

### 3. admin, editor 같은 일반 이름으로 너무 많은 콘텐츠를 발행한다

실제 저자성이 흐려집니다.

### 4. E-E-A-T를 말하면서 작성자 정보를 전혀 업데이트하지 않는다

예전 한 줄 소개만으로는 신뢰가 쌓이지 않습니다.

## 지금 작성자 페이지를 점검하고 싶다면 이 순서가 좋다

### 1단계: 작성자를 분류한다

최소한 다음처럼 나누세요.

- 핵심 작성자
- 가끔 쓰는 작성자
- 게스트 작성자
- 내부 계정

### 2단계: 각 페이지에 독립 가치가 있는지 본다

다음 항목을 확인하세요.

- 글 수
- 주제 일관성
- 작성자 정보의 완성도
- 무엇에 강한 사람인지 명확한지

### 3단계: 템플릿이 너무 얇지 않은지 본다

- 제대로 된 소개가 있는지
- 대표 콘텐츠가 있는지
- 전문성 신호가 있는지
- 다음 읽을 경로가 있는지

### 4단계: 사이트 신호를 맞춘다

- sitemap에 넣어야 할 작성자 페이지는 무엇인지
- 강한 내부링크를 줄 페이지는 무엇인지
- 팀/소개 페이지와 충돌하지 않는지
- 내부 계정을 불필요하게 공개하고 있지 않은지

### 5단계: 실제 결과를 본다

중요한 건 작성자 페이지 수가 아니라:

- 신뢰도가 올라가는지
- 얇은 페이지 부담이 늘지 않는지
- 같은 작성자의 다른 콘텐츠로 자연스럽게 이어지는지

## 마지막 한마디

작성자 페이지의 핵심은 CMS가 만들 수 있느냐가 아닙니다.

진짜 어려운 부분은:

- 어떤 사람을 공개 입구로 만들 것인지
- 어떤 페이지가 실제 신뢰를 만드는지
- 다른 소개성 페이지와 어떻게 역할을 나눌지
- “작성자 페이지”라고 부를 만큼 충분한 정보가 있는지

**모든 계정을 인덱스 페이지로 만들지 마세요. 먼저 사이트 콘텐츠를 진짜 대표하는 작성자를 고르고, 그 사람들의 페이지를 명확하고 유용하고 신뢰할 수 있게 만드는 게 먼저입니다.**
