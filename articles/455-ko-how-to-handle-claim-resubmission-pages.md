## 왜 그렇게 많은 해외 ​​전자상거래 사이트에서 실제로 순위를 매겨야 하는 공개 도움말 콘텐츠 대신 다시 제출 페이지, 다시 제출 URL 및 계속 제출 페이지를 노출하게 됩니까?

나는 최근에 이것을 많이 보았습니다.

많은 팀이 제품 페이지, 카테고리 페이지, 블로그 및 FAQ에 실시간으로 시간을 보냅니다. 그런 다음 청구, 항소, 보충 자료 및 판매 후 흐름이 더욱 복잡해지면 시스템은 다음과 같은 URL을 생성하기 시작합니다.

- `/청구/다시 제출`
- `/청구/다시 제출`
- `/appeal/resubmit?id=xxx`
- `/case/retry-submit`
- `/after-sales/continue-submit`
- `/판매자/청구/재제출-기록`

그리고 첫 번째 반응은 일반적으로 꽤 이해하기 쉽습니다.

- 사용자는 실제로 "청구서를 다시 제출하는 방법"을 검색합니다.
- 페이지에 버튼과 단계, 메모가 있어서 얇아 보이지 않습니다.
- 시스템이 이미 생성했기 때문에 롱테일 트래픽을 포착할 수도 있습니다.
- 거부된 페이지나 닫힌 페이지에 비해 다시 제출한 페이지가 더 유용하다고 느껴지므로 색인을 생성할 가치가 있을 수도 있습니다.

그 논리는 처음에는 합리적으로 들립니다.

그러나 사이트가 충분히 오래 운영되면 이러한 페이지는 종종 실제 SEO 혼란이 됩니다. 하나의 주문, 하나의 사례, 하나의 계정, 하나의 반품 이유 및 하나의 활성 마감일에 크게 의존합니다. 검색 사용자는 해당 페이지에 접속하여 "더 많은 문서를 업로드하고 다시 제출", "이 사례는 한 번만 다시 제출할 수 있습니다", "현재 계정만 계속할 수 있습니다" 또는 "다시 제출 기간이 만료되었습니다"와 같은 내용을 확인합니다.

따라서 검색 엔진은 결국 개인 작업 흐름 URL을 선택하게 되지만 "청구서를 다시 제출하는 방법", "다시 제출하는 데 걸리는 시간" 또는 "반품 후 준비할 사항"과 같은 쿼리에 대해 실제로 순위를 매겨야 하는 페이지는 제대로 구축되지 않습니다.

**핵심 사항입니다. 대부분의 청구서 재제출 페이지는 공개 검색 랜딩 페이지가 아닌 워크플로 페이지입니다. 운영, 지원 및 사용자 완성에 중요하지만 대부분의 경우 중요한 SEO 페이지로 취급해서는 안 됩니다. 실제 질문은 페이지에 버튼이 있는지 또는 다음 단계가 있는지가 아닙니다. 실제 질문은 URL이 재사용 가능한 공개 검색 의도에 응답하는지, 아니면 하나의 비공개 사례에만 서비스를 제공하는지 여부입니다.**

## 재제출 페이지는 실제로 어떤 작업을 수행하나요?

### 1. 이미 청구 또는 항소 흐름에 있는 사람들에게 주로 서비스를 제공합니다.

일반적으로 사용자에게 다음 사항을 알려주는 재제출 페이지가 있습니다.

- 사건을 다시 제출할 수 있는지 여부
- 누락된 문서, 스크린샷, 청구서 또는 설명
- 더 많은 자료를 업로드해야 하는지, 필드를 편집해야 하는지, 지원팀에 문의해야 하는지 여부
- 케이스가 현재 어떤 단계에 있는지, 또 다른 제출이 여전히 허용되는지 여부

즉, 처음 검색하는 방문자가 아닌 이미 컨텍스트를 알고 있는 사람들에게 서비스를 제공하는 것입니다.

### 2. 지위, 정체성, 한계, 타이밍과 밀접하게 연관되어 있습니다.

대부분의 재제출 페이지에는 다음과 같은 내용이 포함됩니다.

- 주문 번호
- 청구 번호
- 반품 사유 또는 정정 사유
- 업데이트가 필요한 필드
- 남은 시도 횟수
- 재제출 기한
- 현재 검토 단계
- 현재 계정 권한

이것이 바로 이러한 페이지가 안정적인 공개 콘텐츠가 아닌 이유입니다.

### 3. 비즈니스 가치는 있지만 자동으로 SEO 가치는 없음

팀이 종종 혼동되는 곳입니다.

그들은 다음과 같이 가정합니다.

- 페이지는 작업 흐름에서 중요하므로 SEO에서도 중요해야 합니다.
- 페이지에 텍스트, 양식 및 버튼이 있으므로 색인을 생성할 가치가 있습니다.
- 그 과정에서 사용자가 봐야 하므로 검색엔진도 봐야 한다.

그러나 사용자가 실제로 검색하는 내용은 일반적으로 다음과 같습니다.

- 반품 후 클레임을 다시 제출하는 방법
- 재제출 전 필요한 서류는 무엇인가요?
- 재제출 후 결과가 나오는 데 걸리는 시간
- 재제출, 재개설, 항소의 차이점
- 시스템에서 다시 제출할 수 없는 이유

이는 하나의 개인 사례 화면이 아닌 공개 가이드, FAQ, 도움말 콘텐츠 및 규칙 페이지에서 더 잘 처리됩니다.

## 청구 재제출 페이지 처리 방법: 먼저 이 5가지 상황을 구분하세요.

### 1. 표준 재제출 페이지, 계속 제출 페이지 또는 수정 후 제출 페이지인 경우 일반적으로 핵심 SEO 페이지가 되어서는 안 됩니다.

이것은 가장 일반적인 경우입니다.

일반적으로 이러한 페이지는 다음과 같습니다.

- 로그인이 필요합니다
- 청구 번호 또는 계좌 신원에 따라 다름
- 구체적인 사례 하나를 설명하세요.
- 리뷰, 업로드, 시도가 변경됨에 따라 계속 변경됩니다.
- 외부 사용자에게는 안정적인 읽기 값을 거의 제공하지 않습니다.

쉽게 말하면 **표준 재제출 페이지는 공개 콘텐츠 페이지가 아닌 워크플로 페이지입니다.**

### 2. 실제 검색 값이 "재제출 방법" 또는 "재제출 전 준비 사항"인 경우 비공개 재제출 페이지에 해당 의도가 전달되지 않도록 하세요.

여기서 많은 팀이 잘못된 결정을 내립니다.

예, 사용자는 "다시 제출", "더 많은 자료 업로드" 및 "계속할 수 없는 이유"와 같은 용어를 검색합니다. 그러나 이것이 비공개 다시 제출 페이지 자체의 순위를 매겨야 한다는 의미는 아닙니다.

사용자가 일반적으로 원하는 것은 다음이 아닙니다.

- 특정한 경우에 누락된 스크린샷
- 오늘 한 계정에서 시도 횟수가 몇 번이나 남았나요?
- 이번에는 하나의 내부 유효성 검사 규칙이 실패했는지 여부
- 하나의 특정 계정에 하나의 특정 버튼이 활성화되어 있는지 여부

그들이 실제로 원하는 것은 다음과 같습니다.

- 일반적으로 청구서를 다시 제출하는 방법
- 일반적으로 필요한 서류는 무엇입니까?
- 버튼이 비활성화된 이유
- 다음 검토에 걸리는 시간
- 재시도를 중단하고 대신 지원팀에 문의해야 하는 경우

이는 공개 설명 페이지, 도움말 페이지, FAQ 및 규칙 페이지에 속합니다.

### 3. 시스템에서 다시 제출, 재시도-제출, 계속-제출, 초안-복구 및 재제출-기록 URL도 생성하는 경우 이를 함께 제어합니다.

실제 문제는 일반적으로 하나의 `/claim/resubmit` URL이 아닙니다.

다음과 같은 클러스터인 경우가 더 많습니다.

- `/청구/다시 제출`
- `/claim/re-submit?case=xxx`
- `/appeal/continue-submit`
- `/case/retry-submit`
- `/claim/draft-recovery`
- `/판매자/청구/재제출-기록`
- `/청구/재제출-성공`
- `/support/case/다시 열고 제출`

그런 일이 발생하면 제목이 겹치고 템플릿이 반복되며 성공 페이지, 초안 페이지 및 레코드 페이지와 같은 얇은 페이지가 아무 이유 없이 색인으로 누출되기 시작합니다.

### 4. 공개 방법 페이지와 비공개 재제출 페이지가 모두 있는 경우 완전히 분리하세요.

많은 사이트가 페이지 수가 너무 적기 때문이 아니라 페이지 역할이 모호하기 때문에 지저분해집니다.

한 페이지 세트는 공개입니다. 예를 들면 다음과 같습니다.

- 청구서를 다시 제출하는 방법
- 재제출 전 준비할 사항
- 재제출 실패 시 대처 방법
- 재제출, 재개설, 항소의 차이점
- 결과를 다시 제출한 후 시간이 얼마나 걸립니까?

또 다른 세트는 비공개입니다. 예를 들면 다음과 같습니다.

- 하나의 주문에 대한 다시 제출 입력 페이지
- 판매자 백엔드의 재제출 내역 페이지
- 추가 업로드 후 계속 제출 페이지
- 한 계정에 대한 검증 실패 재시도 페이지

이 두 세트가 혼합되면 검색 엔진은 잘못된 페이지를 찾고 사용자는 검색한 질문에 완전히 대답하지 못한 화면에 도달하게 됩니다.

### 5. 다시 제출한 페이지의 순위를 지정하지 않으려면 NOINDEX, 로그인 제한, 사이트맵, 표준 및 내부 링크를 일관되게 유지하세요.

많은 재제출 페이지 SEO 문제는 혼합된 신호에서 발생합니다.

- 템플릿에는 NOINDEX라고 표시되어 있지만 사이트맵에는 여전히 다시 제출 URL이 나열되어 있습니다.
- 메인 페이지에는 로그인이 필요하지만 성공, 기록 또는 초안 페이지는 공개 상태로 유지됩니다.
- 다시 제출 페이지, 반환 페이지, 수정 페이지 사이의 표준 태그 지점
- 계정 센터, 도움말 센터 또는 내부 검색이 마치 콘텐츠 페이지인 것처럼 계속 연결됩니다.

재제출 페이지가 주요 SEO 진입 페이지가 되어서는 안 된다는 것을 이미 알고 있다면 문제의 절반만 수정하지 마세요.

더 안전한 설정은 일반적으로 다음과 같습니다.

- NOINDEX를 일관되게 적용하거나 로그인 뒤에 전체 흐름을 유지합니다.
- 사이트맵에서 워크플로 URL을 제외하세요.
- 임시 상태 페이지를 가리키는 표준 태그를 피하세요.
- 다시 제출한 페이지를 내부 링크의 콘텐츠 페이지로 표시하지 마세요.
- 성공, 초안, 기록 및 매개변수 URL이 조용히 크롤링 가능한 진입점이 되는 것을 중지합니다.

## 내가 가장 자주 보는 재제출 페이지 SEO 실수 4가지

### 1. 사용자가 조치를 취할 수 있다는 이유만으로 페이지에 색인을 생성할 가치가 있다고 가정

사용자는 "다시 제출하는 방법"을 검색할 수 있지만 이것이 하나의 비공개 다시 제출 URL이 순위를 매겨야 한다는 의미는 아닙니다.

### 2. 페이지 다시 제출, 페이지 재시도, 페이지 반환 및 다시 열기 페이지를 하나의 키워드로 혼합합니다.

이러한 페이지 유형은 동일한 것을 의미하지 않습니다. 제목, 템플릿 및 내부 링크가 모두 혼합되어 있으면 일반적으로 더 나은 적용 범위 대신 키워드가 중복됩니다.

### 3. 초안, 성공, 기록 및 단계 페이지가 색인으로 유출되도록 허용

이러한 페이지는 내부 워크플로에 도움이 될 수 있지만 일반적으로 검색 값 대신 크롤링 낭비와 중복 콘텐츠를 추가합니다.

### 4. 순위를 매겨야 하는 공개 가이드 구축 실패

이것이 많은 사이트의 근본 문제입니다. 사용자 질문이 존재합니다. 그들은 결코 적절한 공개 콘텐츠로 바뀌지 않습니다.

## 지금 정리하고 싶다면 빠른 감사 명령

### 1단계: 모든 재제출 관련 URL 유형 매핑

최소한 다음을 확인하세요.

- 페이지 다시 제출
- 계속 제출 페이지
- 재시도-제출 페이지
- 초안 복구 페이지
- 성공/기록 페이지
- 매개변수화된 다시 제출 URL
- 계정 센터 또는 판매자 백엔드 재제출 흐름

### 2단계: 공개 콘텐츠 페이지에 속하는 검색 의도 결정

다음과 같은 쿼리에 집중하세요.

- 청구서를 다시 제출하는 방법
- 재제출 전 준비할 사항
- 시스템이 계속 진행을 허용하지 않는 이유
- 재제출 vs 항소
- 재제출 후 시간이 얼마나 소요되나요?

### 3단계: 공개 설명 페이지와 비공개 워크플로 페이지 분할

검색 수요에 대한 공개 가이드, FAQ 및 도움말 페이지를 구축하세요. 사례별 페이지를 워크플로 페이지로 관리합니다.

### 4단계: 한 지점만 고정하는 대신 기술 신호를 통합합니다.

NOINDEX, Canonical, 사이트맵, 로그인 제한, 내부 링크 경로를 함께 살펴보세요.

### 5단계: 다시 제출 페이지가 여전히 나타나는지 여부가 아니라 올바른 페이지의 순위가 매겨졌는지 측정합니다.

당신이 정말로 보고 싶은 것은:

- 검색 트래픽을 수집하는 공개 설명 페이지
- 다시 제출하고, 기록하고, 초안 페이지를 점차적으로 색인에서 제거합니다.
- 제품, 카테고리, 블로그 및 도움말 페이지로 돌아가는 크롤링 예산
- 유용한 답변을 더 빠르게 찾는 방문자 검색
